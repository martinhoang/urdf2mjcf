Current diagnosis

The package has useful capability, but weak failure boundaries.

The new Python API exposes the converter to embedded callers. That makes previously CLI-only behavior, especially logging and returning None on failure, part of a public contract that must be fixed.

Highest-impact bug: several conversion failures are printed and swallowed. API callers cannot reliably distinguish success, partial output, and failure. Correct error propagation is the first task.
Prioritized backlog

Four levels, ordered by dependency.

Each level creates the foundation needed by the next one.

P0 · Correctness

Define the public contract

  • Structured ConversionError with failing stage
  • Non-zero CLI exit codes and API exceptions
  • Typed ConversionResult with all artifacts
  • Shared validation for CLI, JSON, and Python
  • Reject unknown keys and invalid option combinations
  • Resolve relative paths from the config location
P1 · Architecture

Make stages independently testable

  • Typed canonical configuration model
  • Explicit xacro, URDF, mesh, compile, and save stages
  • Stable converter facade
  • Progress callbacks and quiet mode
  • Golden MJCF fixtures and CLI/API parity tests
  • Lint, format, typing, and multi-version CI
P2 · Mesh pipeline

Improve determinism and throughput

  • Content-addressed mesh cache
  • Never mutate source assets
  • Atomic staging and output publication
  • Deterministic names plus asset manifest
  • Bounded workers and memory-aware defaults
  • Benchmarks for representative robots
P3 · Product UX

Add workflows built on stable APIs

  • validate and inspect subcommands
  • Python dry_run=True
  • Machine-readable conversion report
  • JSON schema and editor completion
  • Package resolver hooks
  • Named conversion profiles
Docs · Release

Remove contract drift

  • Generate config template from canonical defaults
  • Verify README flags against CLI help
  • Document API return and exception behavior
  • Version consistency and release checks
  • Minimal versus full dependency extras
Deferred

Keep scope under control

  • Desktop GUI
  • Embedded 3D preview
  • Automatic physical tuning
  • Cloud conversion service

Revisit only after schema, validation, and output determinism stabilize.

Delivery sequence

Small releases with observable outcomes.

Harden errors and configuration

Introduce result/error types, canonical validation, exit codes, relative-path rules, and failure regression tests.

Extract conversion stages

Move behavior behind explicit stage interfaces while preserving the existing CLI and convert(...) facade.

Make assets deterministic

Add staging, manifests, caching, deterministic names, source asset protection, and mesh benchmark coverage.

Build user-facing workflows

Add validation, inspection, dry-run, reports, schemas, and profiles using the stable core rather than duplicating conversion logic.
